Solution

In my own words, the purpose of the set.seed() function is to be able to reproduce random numbers. Part b (Code: 1 pt)

Now we are ready to turn these lines of code into a function. Follow the commented instructions in the chunk below to create the z_score function.

Solution

z_score <- function(x){
xbar <- mean(x)
s <- sd(x)
z <- (x-xbar)/s
return(z)                    # instead of print
}

Part c (Testing: 0.5 pts)

Now let’s test the function.

If you did everything correctly, then R will output the correct z-scores of approximately -0.525, -0.131, 1.444, and -0.788. If you didn’t get those numbers, make sure that you replaced all instances of my_vector inside your function with x.

Even though we didn’t pass my_vector to the function, it’s still hanging out in the Global Environment (see the Environment panel). Since there’s no my_vector inside the function, R will guess that it’s supposed to use the my_vector in the Global Environment.

Environments are a weird and advanced topic, and I point this issue out specifically because it’s the number two cause of Rendering issues (behind code errors). When you run the chunks, R looks for variables in the Global Environment. But when you Render the document, R creates an entirely new Environment that doesn’t have access to anything in the Global Environment. If everything works when you run the code chunks individually but you get errors when you Render the document, it’s very likely that there’s a variable hanging out in your Global Environment that isn’t defined in the document. Click the broom icon in the Environment panel to delete everything from your Global Environment, then rerun your code chunks to see which variable(s) you forgot to define in the document.
